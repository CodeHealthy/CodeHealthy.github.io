# Muhammad Yeshar Portfolio

A responsive static portfolio prepared for GitHub Pages.

## Included

- `index.html` - one-page portfolio
- `styles.css` - responsive UI with dark/light theme
- `script.js` - navigation, scroll reveal, active sections, theme toggle, copy email
- `assets/Muhammad-Yeshar-Resume.pdf` - downloadable resume
- `assets/favicon.svg` and `assets/og-card.svg`
- `.nojekyll` - prevents GitHub Pages from treating the site as a Jekyll project

## Publish to GitHub Pages

### Option A: User site, recommended

1. Create a GitHub repository named exactly:

   ```txt
   CodeHealthy.github.io
   ```

   If your GitHub username changes, replace `CodeHealthy` with the exact username casing GitHub expects.

2. Upload all files from this folder to the repository root.
3. Commit to the `main` branch.
4. Open repository Settings > Pages.
5. Set Source to Deploy from a branch.
6. Set Branch to `main` and folder to `/root`.
7. Your site should become available at:

   ```txt
   https://CodeHealthy.github.io
   ```

### Option B: Project site

Use any repository name, for example `portfolio`, then publish from `main` / `/root`.
The URL will be similar to:

```txt
https://CodeHealthy.github.io/portfolio/
```
